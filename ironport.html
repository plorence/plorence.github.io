IronPort

The IronPort user interfaces maintained a stripped-down, Spartan aesthetic. Part of this was due to the nature of IronPort's target audience of no-nonsense system administrators buying and configuring a dedicated, on-premises security appliance designed for stopping spam and viruses. Another part was due to the fact that the web application was built in a python-based front-end framework hosted from the appliance itself (an on-premises solution vs. a SaaS web-based application). Nearly all of the UI components were built from scratch. Like the Agari products, the IronPort appliances won several awards from SC Magazine and were praised for their ease-of-use.

The UI contained many of the elements expected in modern web-based SaaS application: pages for configuration, a method for building policies, views and permissions for different user roles, dashboards, end-user portals, and a robust reporting system providing real-time, scheduled, and PDF reports.

We licensed Splunk for log file searching, and the UI was localized into 9 languages. The design's utilitarian HTML allowed us to create high-fidelity wireframes in the final presentation medium, which saved time and kept the presentation and the design system under our control. The consistency of the standards allowed us to quickly create similar interfaces for the web security (proxy) appliances and centralized quarantine management appliances as the product line evolved.
